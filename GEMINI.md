# GEMINI.md - CS Fundamentals & Interview Practice

This repository is dedicated to mastering computer science fundamentals, system design principles, and technical interview preparation. It serves as a structured environment for implementing algorithms, exploring design patterns, and documenting core engineering concepts.

## Project Structure

- `coding-prct/`: Java implementations of common interview problems and patterns (e.g., Two Pointers, Sliding Window).
- `notes/`: Comprehensive guides on system design, database internals, networking, and API design.

## Technical Standards & Conventions

### Java Development (`coding-prct/`)
- **Naming:** Follow standard Java camelCase conventions. Classes should have descriptive names reflecting the problem or pattern.
- **Testing:** Every solution must include a `main` method with test cases or a corresponding JUnit test in `src/test/java`.
- **Complexity:** Always document the Time and Space complexity at the top of the solution file.
- **Patterns:** Group problems by their algorithmic pattern (e.g., `com.arrays.two_pointers`) rather than just the company name where possible.

### Documentation (`notes/`)
- **Format:** Use structured Markdown with clear headings and bullet points for readability.
- **First Principles:** When documenting concepts (like Latency or Throughput), focus on the "why" and the underlying physics of software as established in existing notes.

## Workflow

1. **Research:** Analyze the problem or concept. Identify the core pattern.
2. **Implementation:** Write clean, idiomatic code with inline comments explaining non-trivial logic.
3. **Verification:** Validate solutions against multiple edge cases (null inputs, empty collections, large datasets).
4. **Refinement:** Ensure the solution is optimized for the best possible time/space complexity.
